## 9.1. Creación de bases de datos en Oracle

Lo primero que haremos es conectar el servidor con el cliente.  
Verificamos que el servidor esté corriendo; esto es simple: vamos al icono de la cruz, pinchamos y listamos todos los procesos para comprobar que ServiceXE y el TNS Listener están en ejecución. Si esto está bien, el servidor está corriendo.

Nota: Si no ves el icono, debes hacerlo por consola.

Para finalizar la conexión recuerda configurar lo siguiente:

- Host: normalmente `localhost`
- Puerto: normalmente `1521`
- Servicio: `XE`
- USUARIO: SYSTEM
- CONTRASEÑA: root

Recuerda que esta es la clave y la contraseña que en su momento asignaste al servidor; además debes indicar el puerto.

Lo ideal es que marques "guardar contraseña" para que no te la vuelva a pedir.

Si hemos seguido todos los pasos correctamente, ya tendrás conexión a la base de datos con todos los privilegios.

## Creación de bases de datos

Mediante comando podemos escribir:

CREATE TABLE nuevatabla( nombre varchar(15), apellido varchar(15), edad varchar(5) );

Luego seleccionamos todo y pinchamos en el botón verde de la izquierda para ejecutar el comando.

Para comprobar que la tabla se creó correctamente, verifica en la columna de la izquierda: debe aparecer el nombre de la tabla que has creado.

## 9.2. Creación de BBDD en MySQL

En este caso hay que hacer lo mismo que hicimos en Oracle:

- Iniciar correctamente MySQL Workbench.
- Comprobar que el servidor está corriendo.
- Codificar la base de datos.
- Guardar y verificar que esté creada.

Recordemos que para abrirlo debemos iniciar MySQL Workbench; al hacerlo nos dará la opción de conectarlo con el servidor. Al hacer clic en la referencia del servidor se abrirá una pestaña para ingresar el usuario y la contraseña; si luego queremos ver el estado del servidor podemos hacer clic en Server Status.

Al igual que Oracle Workbench, cuenta con un editor donde colocaremos el código SQL y, al ejecutarlo, podemos verificar que la base de datos esté creada buscando el nombre de dicha base de datos.
